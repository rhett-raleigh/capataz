# Docs

Reference documentation for capataz — how the system works, how to set it up,
and how to extend it. These are durable explanations meant to be read by
contributors and agents.

For living project knowledge (decisions, ideas, work plans), see `project/`.

## Contents

- `overview.md` — what capataz is and how the engine/data split works
- `setup.md` — first-time installation and configuration
- `extending.md` — how to add skills and capabilities
- `invocation.md` — how to invoke the orchestrator
- `memory-model.md` — how memory and logging work
- `secrets.md` — secret management and the privacy boundary
- `sessions.md` — session lifecycle and worktree isolation

## Convention

Update docs when the behavior they describe changes — stale docs are worse than
no docs. Keep content generic and shareable; machine-specific details belong in
`config/`.
