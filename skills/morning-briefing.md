# morning-briefing

## When to use
The input asks for a morning briefing, daily summary, day plan, standup prep,
or "what's on my plate." Also triggered by a scheduled daily run.

## Inputs
- Optionally: a date (defaults to today).
- Implicitly: the active profile from `config/`, repo map, tracker config.

## Steps

### 0. Prepare
- Set `DATE` to today (or the requested date).
- Create `data/briefings/` if it doesn't exist.
- Load `memory/learned.md` if it exists (for context on ongoing work).
- Load `memory/log.md` tail (last 20 entries) if it exists (for in-flight threads).
- Read `config/repo-map.md` if it exists (for git activity).
- Read `config/tracker.md` if it exists (for tracker queries).

### 1. Git activity (available now via `gh`)
Check each repo listed in `config/repo-map.md`:
- **Your open PRs**: `gh pr list --author @me --state open --repo <repo>`
- **PRs requesting your review**: `gh pr list --search "review-requested:@me" --state open --repo <repo>`
- **Recent merged PRs** (last 24h): `gh pr list --state merged --search "merged:>YESTERDAY" --repo <repo>`
- **Issues assigned to you**: `gh issue list --assignee @me --state open --repo <repo>`

If `config/repo-map.md` is missing or empty, note: "Git activity skipped —
no repos configured in config/repo-map.md."

### 2. Slack activity (requires MCP:slack)
Check if the Slack MCP is available. If so:
- Unread messages in configured channels (from `config/briefing.md` or a
  sensible default like the last 12 hours).
- Threads where you were mentioned or replied.
- DMs with unread messages.
- Summarize each thread in one line: who, topic, whether it needs action.

If MCP:slack is not available, note: "Slack activity skipped — Slack MCP
not configured. Run `claude mcp add slack ...` to enable."

### 3. Tracker (requires MCP:linear or MCP:atlassian)
Check if a tracker MCP is available (read `config/tracker.md` for which one).
If so:
- Tickets assigned to you, sorted by priority.
- Tickets updated in the last 24 hours.
- Sprint/cycle progress if applicable.

If no tracker MCP is available, note: "Tracker activity skipped — no
tracker MCP configured."

### 4. Calendar (requires MCP:google-calendar or similar)
Check if a calendar MCP is available. If so:
- Today's meetings: time, title, attendees, any linked docs.
- Flag meetings that start within 2 hours for prep.

If no calendar MCP is available, note: "Calendar skipped — no calendar
MCP configured."

### 5. In-flight capataz threads
Review `memory/log.md` tail for recent orchestrator activity:
- Tasks created but not yet resolved.
- PRs opened but not yet merged.
- Threads where the orchestrator is waiting on confirmation.

If `memory/log.md` doesn't exist or is empty, note: "No in-flight threads
(memory/log.md is empty)."

### 6. Goals
Scan `data/goals/*.md` for files with `type: goal` in frontmatter; filter to
`status: active`. Read the current month's `data/goals/log/YYYY-MM.md` (plus
the previous month's if we're in the first days of a month):
- **Streak status per active goal**: consecutive days/weeks with entries,
  computed from the log per the goal's `cadence` field.
- **Overdue flag**: any active goal with no log entry in > 2× its cadence
  (e.g. daily cadence, nothing for 3 days) — flag it explicitly.
- **One priority nudge**: the single most useful goal action for today
  (the most overdue goal, or a streak worth protecting). One line, not a list.

Degrade gracefully: if no `type: goal` files exist in `data/goals/`, note:
"Goals skipped — no goal files found in data/goals/." If goals exist but
`data/goals/log/` has no entries yet, note: "Goals defined but no check-ins
logged yet — say 'goals check-in' to start."

### 7. Synthesize and write
Compile all sections into a structured briefing. Write it to
`data/briefings/YYYY-MM-DD.md` with this structure:

```
# Daily Briefing — YYYY-MM-DD

## Suggested priorities
1. <highest-priority item based on all sources>
2. ...
3. ...

## Git activity
<from step 1>

## Slack highlights
<from step 2, or "not configured">

## Tracker
<from step 3, or "not configured">

## Calendar
<from step 4, or "not configured">

## In-flight threads
<from step 5, or "none">

## Goals
<from step 6: streaks, overdue flags, one priority nudge — or "not set up">

## Unconfigured sources
<list any data sources that were skipped and how to enable them>

---
Generated by capataz morning-briefing skill.
```

### 8. Report back
- If invoked interactively or via Slack: reply with the **Suggested priorities**
  section and a one-line summary of each other section. Link to the full
  briefing file.
- If invoked via scheduled/async run: the file is the deliverable.
- Append a log entry to `memory/log.md`: date, `briefing`, file path.

## Output / report-back
A markdown file at `data/briefings/YYYY-MM-DD.md` and a conversational summary
of priorities and highlights.

## Safety
- Read-only against all external sources (Slack, trackers, calendar, git).
- Never modify tickets, threads, or PRs — only read and summarize.
- Write only to gitignored paths (`data/briefings/`, `memory/log.md`).
